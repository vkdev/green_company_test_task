# Реализация
# Описание проекта GreenTest (Android)

## Исходное задание

Приложение должно:

1. Содержать экран со списком установленных приложений.
2. По нажатию на элемент списка переходить на экран с подробной информацией об установленном приложении.
3. На экране с подробной информацией отображать:
    * название приложения;
    * версия;
    * имя пакета приложения;
    * контрольная сумма APK-файла.
4. На экране с информацией о приложении должна быть кнопка, открывающая приложение.

Алгоритм контрольной суммы можно выбрать самостоятельно (SHA-1, MD5, CRC32 или иной).  
Ограничений по стеку технологий нет. Приложение должно работать на устройствах с ОС Android 7.0 и выше.

---

## Реализованный функционал

### Экраны

- **Список установленных приложений**  
  Отображаются все установленные приложения: иконка, название и имя пакета. Реализована опция «только с возможностью запуска» (чекбокс): можно показывать только те приложения, у которых есть launch activity (их можно открыть). По нажатию на элемент выполняется переход на экран деталей.

- **Подробная информация о приложении**  
  Показываются:
    * название приложения;
    * версия (versionName);
    * код версии (versionCode);
    * имя пакета;
    * контрольная сумма APK (SHA-256);
    * иконка приложения;
    * кнопка «Открыть приложение» — отображается только если у приложения есть launch activity (его можно запустить).

Контрольная сумма вычисляется по APK-файлу приложения (`sourceDir`). Выбран алгоритм **SHA-256**. Расчёт выполняется асинхронно после загрузки базовых данных, чтобы экран быстрее отображался; хеш подставляется на экран по мере готовности.

### Дополнительные особенности

- Обработка ошибок: при ошибке загрузки данных или приложения показывается сообщение об ошибке.
- Кэширование иконок приложений (LRU) для снижения нагрузки при прокрутке списка.
- Поддержка Android 7.0 и выше: `minSdk = 24`, `targetSdk = 36`.

---

## Стек технологий

- **Язык:** Kotlin
- **UI:** Jetpack Compose, Material 3
- **Навигация:** androidx.navigation3
- **DI:** Koin (Koin Android, Koin Compose, ViewModel)
- **Асинхронность:** Kotlin Coroutines
- **Сборка:** Gradle (Kotlin DSL), AGP 8.x

Сторонние библиотеки для хеширования не используются: контрольная сумма считается через `java.security.MessageDigest` (MD5 и SHA-256 вынесены в отдельный модуль).

---

## Архитектура и структура модулей

Проект разбит на модули по зонам ответственности:

| Модуль | Назначение |
|--------|------------|
| **app** | Точка входа, сборка приложения, общая DI-конфигурация. |
| **ui-common** | Общие UI-ресурсы: тема, цвета, типографика, отступы. |
| **features/feature-applications-list** | Экран списка приложений (экран, ViewModel, DI). |
| **features/feature-applications-list-api** | API экрана списка (фабрика экрана для навигации). |
| **features/feature-app-details** | Экран деталей приложения (экран, ViewModel, запуск приложения, DI). |
| **features/feature-app-details-api** | API экрана деталей (фабрика экрана для навигации). |
| **features/navigation** | Граф навигации (список → детали, переход по packageId). |
| **repository/repository-api** | Интерфейс репозитория и модели данных (AppInfo, HashAlgorithm). |
| **repository/repository-impl** | Реализация: получение списка приложений, данных по пакету, иконок, потока APK для хеша. |
| **hash-function** | Вычисление хеш-сумм (MD5, SHA-256) по потоку; интерфейс расширяемый. |
| **cache/cache-api** | Интерфейс in-memory LRU-кэша. |
| **cache/cache-impl** | Реализация LRU-кэша для Bitmap (иконки). |
| **logging/logging-api** | Интерфейс логгера (info, warn, error с опциональным Throwable). |
| **logging/logging-android** | Реализация логгера на базе `android.util.Log`. |

Экраны не зависят друг от друга напрямую: навигация и передача параметров (например, `packageId`) идут через общий граф и API-модули фич. Данные об приложениях и хеши запрашиваются через репозиторий; хеш считается в фоне с возможностью отмены (корутины). Есть unit-тесты для ViewModel списка, деталей и хеш-функции.
